# /cowork-os:linkedin-editor

Turn a source (article, case study, deck, draft, published post, or rough notes) into ready-to-publish LinkedIn posts. Optimize for impressions, comments, profile visits, and leads, never for likes. Use the workspace's `context/positioning.md` and `context/tone_of_voice.md` if present.

Run these 8 steps:
1. Read the source fully. If it is a link, read it. Identify the core argument and the proof.
2. Extract 20 distinct post ideas from it.
3. Classify each idea by format: story, contrarian take, how-to, case or proof, list, or question.
4. Pick the 5 strongest for this audience and goal.
5. For each of the 5, write 3 hook variants (the first 150 characters that stop the scroll).
6. Score each of the 5 on reach potential and lead potential (low, medium, high) with a one-line reason.
7. Pick today's single best post and write it in full: founder voice, concrete, no hype, no long dash. Put any link in the first comment, not in the body.
8. Deliver: the finished post, the 2 runner-up outlines, and the hook bank.

Source: $ARGUMENTS. If empty, ask for the source before starting.

End with a Memory Update if a content decision emerged (route reusable ideas to `marketing/content.md`).
