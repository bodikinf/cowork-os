# Contributing · Contribuire

> **EN —** This is a structure-and-prompts project, not code. Contributions that keep it simple, generic and adoptable are welcome.
> **IT —** Questo è un progetto di struttura-e-prompt, non codice. Sono benvenuti contributi che lo mantengano semplice, generico e adottabile.

## EN

Good contributions:
- New **template** files or modules that are genuinely reusable across companies.
- Improvements to the **Memory Update** protocol or the workflows.
- Translations and clearer wording.
- New **scheduled-task** templates (keep them generic, no private data).

Rules:
- **Never commit private data** — no real client names, financials, emails, API keys, or CRM IDs. Use `{{placeholders}}`.
- Keep the **template + sanitized example** style: a reusable skeleton, then a short illustrative example.
- Markdown only. No binaries, no `.DS_Store`.
- One module per pull request when you can.

Open an issue first for anything large, so we can agree on the shape.

## IT

Contributi utili:
- Nuovi file **template** o moduli davvero riutilizzabili tra aziende diverse.
- Miglioramenti al protocollo **Memory Update** o ai workflow.
- Traduzioni e formulazioni più chiare.
- Nuovi template di **scheduled-task** (generici, senza dati privati).

Regole:
- **Mai committare dati privati** — niente nomi clienti reali, dati finanziari, email, API key o ID CRM. Usa i `{{placeholder}}`.
- Mantieni lo stile **template + esempio sanitizzato**: scheletro riutilizzabile, poi un breve esempio illustrativo.
- Solo Markdown. Niente binari, niente `.DS_Store`.
- Un modulo per pull request quando possibile.

Per modifiche grandi, apri prima una issue così concordiamo la forma.
